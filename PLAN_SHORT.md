# Короткий план завершения

## Что должно быть внутри
- Минимальный публичный API headless-ядра (движок, события, RNG, эффекты).
- Подключаемые правила и режимы через плагины и конфигурацию.
- Детерминированный цикл хода: draw → main → combat → end.
- Минимальная документация и примеры конфигураций карт/правил.
- Контроль размера ядра и базовые проверки детерминизма.

## Ближайшие шаги
1. Зафиксировать API конфигурации ядра и подключение правил.
2. Описать формат правил/карт и сценарии запуска.
3. Провести аудит размера и убрать лишнее.
